# Tenways Octo 扩展自动更新功能 PRD

## 1. 需求概述

### 1.1 背景
Tenways Octo 是一个非 Chrome Web Store 分发的浏览器扩展，需要实现自托管的自动更新机制，确保用户能够及时获取最新版本的功能和安全修复。

### 1.2 目标
- 实现扩展的静默版本检测
- 通过视觉提示告知用户有新版本
- 提供便捷的更新安装流程

### 1.3 优先级
**P1 - 重要功能**（非阻塞但重要的功能，建议在当前迭代或下一迭代完成）

---

## 2. 功能需求

### 2.1 版本检查机制

#### FR-001: 静默检查
- **描述**: 扩展在后台静默检查服务器是否有新版本
- **触发时机**:
  - 扩展启动时（browser startup）
  - 每 24 小时定时检查一次
  - 用户主动点击扩展 popup 时
- **检查方式**: 向服务器 `/api/extension/version` 发送请求，获取最新版本信息
- **版本对比**: 将服务器返回的版本号与当前 `manifest.json` 中的版本号对比

#### FR-002: 版本信息格式
服务器应返回以下格式：
```json
{
  "version": "0.7.0",
  "downloadUrl": "https://cdn.example.com/tenways-octo-0.7.0-chrome.zip",
  "releaseNotes": "修复了...",
  "forceUpdate": false,
  "minVersion": "0.6.0"
}
```

### 2.2 更新提示机制

#### FR-003: Badge 提示
- **描述**: 当检测到有新版本时，在扩展图标上显示红色数字 badge
- **显示内容**: 显示 `"1"` 或 `"NEW"`（根据 Chrome 限制，badge 最多显示 4 个字符）
- **颜色**: 红色背景 (`#FF0000`)，白色文字
- **清除条件**: 用户点击扩展图标打开 popup 后清除 badge

#### FR-004: Popup 内提示
- **描述**: 在扩展 popup 内显示更新提示横幅
- **显示位置**: popup 顶部
- **显示内容**:
  - 当前版本 vs 新版本号
  - 更新内容摘要
  - "立即更新" 按钮
- **样式**: 醒目的通知栏样式，区别于普通内容

#### FR-005: 浏览器通知（可选）
- **描述**: 对于 forceUpdate = true 的强制更新，显示系统级浏览器通知
- **触发条件**: 仅在需要强制更新时显示
- **权限**: 需要添加 `notifications` 权限到 manifest

### 2.3 更新安装流程

#### FR-006: 下载更新包
- **描述**: 用户点击"立即更新"后，自动下载新版本 zip 包
- **下载方式**: 使用 Chrome `downloads` API 下载到用户默认下载目录
- **下载进度**: 在 popup 内显示下载进度条
- **文件命名**: `tenways-octo-{version}-update.zip`

#### FR-007: 安装引导
由于 Chrome 安全限制，扩展无法自动安装新版本，需要引导用户完成安装：
1. 下载完成后显示安装指引弹窗
2. 引导用户打开 `chrome://extensions/` 页面
3. 提示用户开启"开发者模式"
4. 提示用户将下载的 zip 包拖拽到扩展页面，或使用"加载已解压的扩展程序"
5. 提供"稍后提醒"和"忽略此版本"选项

#### FR-008: 忽略版本
- **描述**: 用户可以选择忽略当前版本更新
- **行为**: 将版本号保存到 `chrome.storage.local`，24 小时内不再提示该版本
- **重置**: 用户可以在扩展设置中清除忽略记录

## 3. 非功能需求

### 3.1 性能要求
- 版本检查请求应在 2 秒内完成
- 不应阻塞扩展的其他功能
- 检查失败时不应影响扩展正常使用

### 3.2 安全要求
- 版本检查请求应使用 HTTPS
- 下载的 zip 包应进行 SHA256 校验（如果服务器提供校验值）
- 不应自动执行任何安装操作

### 3.3 兼容性
- 支持 Chrome 88+ 版本
- 与现有扩展功能兼容，不破坏现有消息路由

## 4. 技术方案

### 4.1 涉及的文件变更
```
extension/
├── src/
│   ├── background/
│   │   ├── update-checker.ts      # 版本检查核心逻辑
│   │   └── index.ts               # 入口点，注册检查任务
│   ├── popup-react/
│   │   ├── components/
│   │   │   └── UpdateBanner.tsx   # 更新提示横幅组件
│   │   └── App.tsx                # 集成更新提示
│   └── types/
│       └── update.ts              # 更新相关类型定义
├── manifest.json                  # 添加 notifications 权限（可选）
└── package.json                   # 添加版本号常量
```

### 4.2 服务器端接口
需要后端提供以下接口：
- `GET /api/extension/version` - 获取最新版本信息
- `GET /api/extension/download/:version` - 下载指定版本 zip 包

### 4.3 权限变更
需要在 `manifest.json` 中添加：
```json
{
  "permissions": [
    "alarms",           // 用于定时检查
    "notifications"     // 可选，用于强制更新通知
  ]
}
```

## 5. 验收标准

### 5.1 功能验收

| 编号 | 验收项 | 验收标准 | 优先级 |
|------|--------|----------|--------|
| AC-001 | 版本检查 | 扩展启动时自动向服务器检查版本，请求成功 | P0 |
| AC-002 | 版本检测 | 服务器版本高于本地版本时，正确识别为新版本 | P0 |
| AC-003 | Badge 提示 | 检测新版本后，扩展图标显示红色数字 badge | P0 |
| AC-004 | Popup 提示 | 打开 popup 后，顶部显示更新提示横幅 | P0 |
| AC-005 | 版本信息展示 | 横幅正确显示当前版本和新版本号对比 | P1 |
| AC-006 | 下载功能 | 点击"立即更新"后，自动下载 zip 包到本地 | P0 |
| AC-007 | 安装引导 | 下载完成后显示清晰的安装步骤指引 | P1 |
| AC-008 | 忽略功能 | 用户可以选择"忽略此版本"，24 小时内不再提示 | P2 |

- 版本检查 API 响应时间 < 2s

- 在 Chrome 88、100、120+ 版本中测试通过
- 与其他扩展功能（Meegle Auth、Lark Auth 等）无冲突

## 6. 风险与注意事项

### 6.1 技术风险
1. **Chrome 限制**: Chrome 不允许扩展自动安装/更新非商店扩展，必须通过用户手动操作
2. **权限变更**: 如果未来需要新权限，自动更新可能无法工作，需要用户重新安装
3. **下载中断**: 大文件下载可能中断，需要考虑断点续传或重试机制

### 6.2 用户体验风险
1. 频繁的更新提示可能打扰用户
2. 手动安装流程对新用户不够友好

### 6.3 缓解措施
1. 提供详细的安装图文/视频教程
2. 允许用户设置检查频率（每日/每周/从不）
3. 只在重要更新时显示强制通知

## 7. 后续迭代

### 7.1 未来优化方向
- **增量更新**: 只下载变更的文件，而非完整 zip 包
- **自动解压**: 提供一键解压脚本，简化安装流程

- **灰度发布**: 支持按用户群体分批推送更新

### 7.2 可能的扩展
- 支持 Firefox 扩展自动更新
- 支持 Edge 扩展自动更新
- 与企业内部扩展商店集成
